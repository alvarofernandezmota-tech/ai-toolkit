## 🔥 Bloque tarde (18:30 — 19:00h) — Investigación de modelos para OpenCode

### Objetivo
Encontrar el modelo principal para OpenCode que aguante contexto largo del repo (~32K tokens).

### Modelos probados y resultados

| Modelo | Proveedor | Resultado | Causa |
|---|---|---|---|
| `llama-3.3-70b-versatile` | Groq | ❌ Límite contexto | 12K TPM, repo necesita 32K |
| `gemini-2.5-flash-preview-04-17` | Google AI | ❌ ID no válido | No existe en v1beta |
| `gemini-2.0-flash` | Google AI | ❌ Rate limit | "too hot" en horas pico (7pm) |
| `gemini-1.5-flash` | Google AI | ❌ ID no válido | No existe en v1beta |
| `gemini-1.5-flash-latest` | Google AI | ⚠️ Pendiente test | No confirmado |
| `qwen3-235b-a22b` | OpenRouter | ❌ Sin créditos | Requiere cuenta de pago |
| `llama3.1-8b` | Cerebras | ⚠️ Parcial | Rate limit TPM en horas pico |

### Aprendizajes tarde

1. **Horas pico = rate limits por todos lados** — las 7pm es hora de máximo uso en todos los proveedores gratuitos
2. **IDs de Gemini son sensibles** — necesitan versión exacta, hacer `curl ListModels` antes de usar
3. **Cerebras funciona** pero con límite 8K de contexto — compacta automáticamente
4. **Groq solo para prompts cortos** — repos grandes lo superan siempre
5. **La solución real es LiteLLM como proxy local** — gestiona fallbacks automáticos entre todos los modelos

### Variable de entorno correcta para Google AI
```bash
# ❌ Incorrecto
export GOOGLE_API_KEY="..."

# ✅ Correcto para OpenCode
export GOOGLE_GENERATIVE_AI_API_KEY="..."
```

### IDs Gemini a verificar con curl (pendiente)
```bash
curl "https://generativelanguage.googleapis.com/v1beta/models?key=$GOOGLE_GENERATIVE_AI_API_KEY" | python3 -m json.tool | grep '"name"'
```

## 🧠 Aprendizajes del día completo

1. **DeepSeek V3-0324 gratis** es excelente para tareas cortas pero se queda sin tokens en sesiones largas
2. **El Compaction** es un resumen interno del contexto — no indica error, solo gestión de memoria
3. **Encadenar demasiadas tareas** en una sola sesión agota el contexto rápido
4. **Mejor estrategia:** tareas de máximo 3 pasos encadenados, luego push y nueva sesión
5. **Todos los proveedores gratuitos tienen rate limits** — la solución es rotación automática con LiteLLM
6. **Los IDs de modelos varían** entre curl directo y OpenCode — siempre verificar con curl primero
7. **Nunca compartir API keys** en chats ni subirlas al repo
